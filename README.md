# Employee Management Microservice (empl-back)

Демонстрационный высокопроизводительный микросервис для управления сотрудниками, отделами и должностями, написанный на современном C++. Проект демонстрирует применение чистой архитектуры (Clean Architecture), контейнеризацию и автоматизированное тестирование.

## 🛠 Технологический стек

* **Язык и сборка:** C++17/20, CMake, CPM (CMake Package Manager)
* **Web-фреймворк:** [Crow](https://crowcpp.org/) (быстрый и легковесный C++ микрофреймворк)
* **База данных:** PostgreSQL
* **Работа с БД:** [libpqxx](https://github.com/jtv/libpqxx) (официальный C++ клиент для PostgreSQL)
* **Миграции:** [dbmate](https://github.com/amacneil/dbmate) (простой и надежный инструмент для миграций)
* **Логирование:** [spdlog](https://github.com/gabime/spdlog) (сверхбыстрая библиотека логирования)
* **Сериализация:** [nlohmann/json](https://github.com/nlohmann/json) (JSON для современных C++)
* **Тестирование:** Google Test (GTest) & Google Mock (GMock)
* **Инфраструктура:** Docker, Docker Compose, Make

## 📂 Структура проекта

Проект организован по принципам чистой архитектуры для обеспечения слабой связности и простоты тестирования:

```text
├── src/                # Исходный код приложения
│   ├── api/            # Контроллеры, DTO и маршрутизация (Crow handlers)
│   ├── core/           # Бизнес-логика, модели, интерфейсы (contracts) и ошибки
│   ├── infrastructure/ # Конфигурация, подключение к БД, фабрика логгеров
│   ├── repository/     # Реализация доступа к данным (libpqxx)
│   └── service/        # Бизнес-сервисы, связывающие API и репозитории
├── tests/              # Модульные (unit) и интеграционные (integration) тесты
├── db/                 # SQL схемы и миграции (dbmate)
├── docker/             # Dockerfile для приложения, тестов и миграций
├── docker-compose.yml  # Оркестрация локального окружения
└── Makefile            # Удобные команды для разработки
